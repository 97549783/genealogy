# Архитектура проекта

## Основная идея

Проект организован по схеме `core/ + tabs/`.

- `tabs/<имя_вкладки>/` — **пакет функциональности** одной вкладки (интерфейс + логика этой вкладки).
- `core/` — только общий код: инфраструктура, общие вспомогательные модули, общий домен.
- `streamlit_app.py` — тонкая **точка композиции**: конфигурация, инициализация и маршрутизация вкладок.

## Правила размещения кода

1. Если код используется только одной вкладкой, он должен находиться в `tabs/<name>/`.
2. В `core/` переносится только код, который:
   - используется минимум в двух вкладках; или
   - является инфраструктурой приложения.
3. Нельзя создавать новый «общий свалочный» модуль вида `core/utils.py`.
4. Плоские устаревшие модули верхнего уровня и переходные `utils/*` удалены намеренно.

## Границы и ответственность

- `core/app/` — инициализация, шапка, секретная административная ветка, контекст приложения.
- `core/db/` — доступ к данным и константы источников данных.
- `core/lineage/` — граф родословной и нормализация имён.
- `core/ui/` — повторно используемые интерфейсные вспомогательные модули и рендереры.
- `tabs/*` — независимые пакеты функциональности вкладок.

## Развитие архитектуры

- Любой перенос кода из вкладки в `core/` должен обосновываться повторным использованием.
- Изменения хранилища (например, CSV → SQLite) должны изолироваться за `core/db`.
- Публичные параметры запроса и данные для ссылок совместного доступа не должны меняться без отдельной миграции.

## Языковое правило

Для единообразия документации комментарии и строки документации в затрагиваемых файлах пишутся на русском языке.
