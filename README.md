# genealogy

Веб-приложение на Streamlit для анализа академических родословных, научных школ, тематических профилей диссертаций и публикационной активности.

## Источник данных

Единственный рабочий источник данных сайта — SQLite-база `genealogy.db` в корне репозитория.

Основные таблицы:

- `diss_metadata` — метаданные диссертаций;
- `diss_scores_5_8` — тематические профили диссертаций;
- `articles_metadata` — метаданные статей;
- `articles_scores_inf_edu` — тематические профили статей.

CSV-файлы не используются runtime-кодом приложения как источник данных. Экспорт результатов пользователем в CSV/XLSX при этом остаётся штатной функцией интерфейса.

Подробнее: [docs/sqlite_data_source.md](docs/sqlite_data_source.md).

## Локальный запуск

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
streamlit run streamlit_app.py
```

По умолчанию приложение ищет базу данных по пути:

```text
genealogy.db
```

Если база находится в другом месте, задайте переменную окружения:

```bash
SQLITE_DB_PATH=/путь/к/genealogy.db streamlit run streamlit_app.py
```

## Структура проекта

- `streamlit_app.py` — точка композиции Streamlit-приложения;
- `core/` — общий код приложения: доступ к данным, граф родословной, общие UI-компоненты, классификаторы;
- `tabs/` — независимые пакеты функциональности отдельных вкладок;
- `docs/` — архитектурная и эксплуатационная документация;
- `genealogy.db` — публичная SQLite-база данных сайта.

## Правила разработки

- Runtime-доступ к данным должен проходить через `core/db/`.
- Вкладки не должны напрямую читать файлы данных.
- Публичные комментарии, строки документации и пользовательские сообщения в затрагиваемом коде пишутся на русском языке.
